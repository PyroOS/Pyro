Bienvenido a Syllable

Agradecemos su confianza en Syllable, uno de los mas modernos y sorprendentes sistemas operativos de hoy! Syllable lleva muchos años en constante desarrollo, ofreciendo un sistema operativo muy compacto, rapido, bien dieñado y muy facil de usar. A diferencia de otros sistemas, solo tenemos un objetivo: Ser el mejor sistema operativo open source en la computadora de escritorio. Para servidores, ofrecemos un producto similar, pero separado: Syllable Server.

El software y los archivos distribuidos con Syllable son publicados bajo distintos copyrights y licencias. El documento de Copyrights tiene mas informacion.

Syllable ya mismo ofrece capacidades internet y multimedia, y su usted tiene interes por el desarrollo de esta tecnologia, lo espera una arquitectura unica y excitante.

Sea lo que fuere, este documento pretende asistirlo en sus primeros contactos son el sistema operativo Syllable, ofreciendo ademas enlaces a mas informacion.

Configurando su lenguaje y teclado

La distribucion regular de Syllable funciona en idioma ingles y toma por defecto un teclado de estados unidos (US). usted puede cambiar el idioma entrando en preferences/locale mediante el menu Syllable o la carpeta Preferences en el desktop o escritorio. Hay una lista de paises e idiomas que se pueden elegir, asi como elegir multiples idiomas. En este ultimo caso, las traducciones de los lenguages mas altos en la lista tendran prioridad sobre los mas bajos. Si no hay traduccion para un menu en un determinado idioma, este sadra en ingles.

Para cambiar la configuracion del teclado, entre en Preferences / Keyboard (en español: preferencias / teclado). Usted puede seleccionar muchos mapas de teclado o cambiar alguna configuracion.

Configurando la red

Syllable usa la tecnologia Dynamic Host Configuration Protocol (DHCP) por defecto para encontrar una conexion de red. No obstante, esto no se refleja por el momento en el panel de control Red / Network. Cuando arranque Syllable, tendra un mensaje en pantalla diciendo que la configuracion de red ha cambiado y debe ser refrendada, en la mayoria de los casos, la red sera detectada mediante DHCP. Para evitar este mensaje, o fijar una red permantente, entre en preferencias/ red y configure manualmente. Si no se muestran interfaces de red, Syllable no reconocio ninguna placa de red en su PC.

Configuring sound

El sonido no esta activado por defecto cuando usted inicia Syllable por primera vez. Para configurar esto entre en Preferencias / Media. Primero debe seleccionar la salida del audio con un menu desplegable (las opciones se muestran todas juntas con el nombre de su chip de audio, si esta vacio es que Syllable no reconoce su chip). Normalmente, deberia elejir Media Server. No obstante, a veces este metodo interfiere con la reproduccion de video. Para mejor video, seleccione la salida de audio OSS. Presione el boton Aplicar para que el cambio tome efecto.

Luego haga click en el boton Stream Volumes para mostrar el audio mixer. La primer pestaña comanda los canales de hardware. Puede tener que subir los niveles de Volumen (line out) y PCM ( audio generado por software) para tener un sonido agradable. El segundo panel es el mixer por software donde los stream de audio son mostrados por separado. Note que el control Master Volume en el mixer de software esta limitado por los controles Volumen y PCM del hardware. Cierre todas las ventanas de control de audio antes de abrir una aplicacion multimedia.

Por favor no olvide que por el momento Syllable no recuerda los niveles configurados de acuerdo a lo descrito arriba depues de un reinicio o un apagado. En el Dock, usted puede colocar un applet con acceso rapido al Mixer.

Accediendo unidades de almacenamiento

Syllable puede acceder particiones extra que pueda tener en su disco o medios externos como Pendrives / MP3 y algunas camaras USB . Medios externos son  Discos FAT (WIndows). Syllable puede leer y escribir en ellos. Los discos tambien pueden estar formateados en BFS (BeOS), NTFS (Windows), Ext2/3. Syllable puede por el momento leer discos NTFS y Ext2/3, pero aun no escribir.

Antes de poder ingresar a un disco, primero debe montarlo. Puede hacer esto abriendo Discos en el escritorio. Luego haga click con el boton derecho en el fondo de la ventana. Seleccione la opcion Montar en la ventana pop-up que aparece para ver una lista de particiones y discos y seleccione cual quiere montar. Un disco / particion montado aparece como un icono extra en la carpeta Discos, de donde puede abrirlo y navegar dentro de el. En el sistema de archivos, este disco se monta en el directorio raiz o Root (/) seguido de un subdirectorio con el nombre de la particion (o /no name/ si no tiene nombre).

Por favor no olvide que los medios montados deben desmontarse de nuevo antes de poder quitarlos con seguridad, como por ejemplo un memory stick. Puede hacer esto mediante un click derecho en el icono de la unidad en cuestion y seleccionar "unmount" o "desmontar". Todos los medios se desmontan automaticamente cuando Syllable es apagado.

Encendido and apagado

Syllable puede ser cerrado y reiniciado seleccionando "Apagar" del menu Syllable en el dock. Todas las aplicaciones reciben la orden de cerrarse correctamente, pero a veces alguna puede impedir el cierre de Syllable. Si esto ocurre, se puede forzar el reinicio del sistema  presionando Control + Alt + Delete / Supr en español. Esto forzara la aplicacion a cerrarse, pero cerrara Syllable ordenadamente, desmontando todos los medios.

Software

Syllable esta diseñado como un sistema de escritorio moderno, y tiene un diseño unico que hace facil el manejo del software.

Basicamente, separamos el software para Syllable en dos categorias. Esta el software escrito especialmente para Syllable, al que llamaremos Nativo, y el software portado de otrosa sistemas como Linux, al que llamaremos Portado.

Como el software portado no puede aprovechar todas las ventajas de Syllable, la instalacion de este software difiere enormemente del software Nativo, que simplemente se debe extraer en algun lado. Debajo estan las instrucciones para cada caso:

Instalando software nativo

Las aplicaciones nativas de Syllable se pueden colocar en cualquier lugar, si bien el mejor es la carpeta /Applications. La mayoria de las aplicaciones vienen como un simple archivo, que solo debe ser desempaquetado donde ud quiera colocar el software. Syllable usa para esto el formato ZIP. En este ejemplo, agregamos el nuevo navegador Webster en la carpeta Aplicaciones. Tenga en cuenta que a la hora de extraer la aplicacion, el comando unzip conoce la carpeta como Applications sin importar que idioma usted use.

unzip Webster-alpha1.application -d /Applications

Eliminar la aplicacion es borrar el archivo con el mouse o bien usar este comando:

rm -r /Applications/Webster

Dejamos en claro que esta solucion es temporal y que se desarrolla un programa estilo WinRar que permitira extraer el paquete con un click dejando totalmente de lado la consola de comandos.

Instalando software portado

Las aplicaciones Unix/Linux/POSIX suelen venir como un paquete que contiene todos los archivos. Syllable uso en su momento el formato Gzip Tar, pero ahora alienta el ZIP.

Para instalar un paquete portado, abra la terminal Aterm, ingrese como usuario Root y desempaque el archivo con un comando como:

unzip gcc-4.1.2-1.i586.resource -d /usr

Luego debe ejecutarse el gestor de paquetes Syllable para registrar este en el sistema. Por ejemplo, para instalar GCC, ejecute:

package register gcc

Esto creara una serie de symlinks en varios subdirectorios de /usr/indexes/ para ahcer posible al sistema encontrar los ejecutables y otros archivos del paquete.

Si quiere remover el paquete, primero debe desregistrarse y despues ser borrado:

package unregister gcc

Despues puede simplemente elimiarlo:

rm -rf /usr/gcc

Por favor note que este procedimiento solo debe ser usado para aplicaciones basadas en texto y portadas desde otros sistemas Unix-like. No intente este procedimiento con las aplicaciones Nativas.

Desarrollando para Syllable

Syllable aun esta bajo fuerte desarrollo, y hay grandes oportunidades para quien este interesado en desarrollar software para Syllable o trabajar con el.

Syllable es un sistema operativo compatible con las normas POSIX, con grandes porciones de POSIX 1,2 y 4 disponibles. Encima de esa capa POSIX, corre un unico set de APIs C++, que proveen clases, interfaz grafica, E/S de alto nivel y componentes de red que pueden tomar ventaja del avanzado diseño de Syllable.

Las herramientas de desarrollo se basan en las usadas por el proyecto GNU, como ser BinUtils, GCC 4 y Make. Hay editores e IDE's para Syllable, disponibles en nuestro sitio web.

Es posible portar gran cantidad de aplicaciones de consola desde sistemas como Unix / Linux, pero Syllable no usa X11 ni Toolkits X. La portacion de aplicaciones graficas requieren una GUI nativa.

Gran parte del kernel y sistemas POSIX estan licenciados bajo los terminos GNU General Public Licence. Las librerias C++ lo estan bajo GNu Lesser General Puiblic Licence

Si usted esta interesado en desarrollar en / para Syllable, no deje de unirse en la Syllable-Developer mailing list. El codigo fuente de Syllable esta disponible via CVS anonimo en http://syllable.cvs.sourceforge.net/syllable/syllable/
